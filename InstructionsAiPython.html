AI Instructions for Writing Python Code

  1. Every function should have a one line docstring
  2. All const should be at the top, below the imports in all caps
  3. Use underscores for variable names
  4. All functions should have a single responsibility
  5. Use is not operator rather than not ... is
  6. Use list comprehensions and generator expressions for simple cases
  7. All functions and variables should have very descriptive names
  8. Add print statements after every action for easier debugging
  9. Use classes and object-oriented programming where appropriate
  10. Follow the PEP 8 style guide for Python code
  11. Write tests for your code to ensure it works as expected
  12. Use comments sparingly and only where necessary. Good code should be self-explanatory
  13. Avoid using global variables as much as possible
  14. Use appropriate data structures for the task at hand
